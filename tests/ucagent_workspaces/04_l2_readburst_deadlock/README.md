# L2ReadBurstDeadlock

This UCAgent workspace demonstrates the same latest-upstream NutShell L2 Cache
wrapper with both formal-skill and official Toffee/pytest flows.

The DUT shell is generated by:

```bash
python3 ../../tests/cases/04_l2_readburst_hit_ready_valid_deadlock/scripts/prepare_latest_l2_readburst.py --repo-root ../.. --timeout 1200
```

The recommended full-demo flow is formal-first:

1. Use `generic-formal` as a pre-diagnosis skill.
2. If formal finds a counterexample, write the reproduction idea.
3. Continue to the original official Toffee/pytest `RunTestCases` flow anyway.

Run it from the repository root with:

```bash
source .ucagent_env
bash scripts/internal/45_run_l2_readburst_ucagent_full_demo.sh
```

The standalone formal-skill flow is:

1. Use `ListSkill` and read `.ucagent/skills/generic-formal/SKILL.md`.
2. Use `RunSkillScript` to run:
   - `tests/cases/04_l2_readburst_hit_ready_valid_deadlock/formal/l2_readburst_assert.yaml`
   - `tests/cases/04_l2_readburst_hit_ready_valid_deadlock/formal/l2_readburst_cover.yaml`
3. Summarize `reports/04_l2_readburst.md` in this workspace.
4. Use `SetSkillUsage` before `Complete`.

Do not run PR21, PR74, MMIO prefetch, idBits, or historical calibration flows in
this workspace.

The standalone Toffee dynamic flow uses the human-curated files under `unity_test/tests`.
For that flow, run `RunTestCases` with:

```text
test_l2_readburst_ready_valid.py -q
```

The Toffee coverage is scenario-level coverage for 04 only, not full NutShell
Cache functional coverage.
